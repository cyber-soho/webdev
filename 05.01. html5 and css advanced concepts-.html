TOPIC 01

HTML5 Semantic Elements

Giving meaning to your markup

Before HTML5 (HyperText Markup Language, version 5), every page was built from generic <div> tags — hundreds of them. Screen readers, search engines, and future-you all saw nothing but anonymous boxes. HTML5 introduced semantic elements — tags whose names describe their purpose.

Semantic Tags Reference

Semantic TagWhat It ReplacesPurpose
<header><div id="header">Page or section banner
<nav><div class="navigation">Navigation links
<main><div id="content">Primary content (only one per page)
<article><div class="post">Self-contained content (blog post, news story)
<section><div class="group">Thematic grouping of related content
<aside><div class="sidebar">Tangentially related content (sidebar)
<footer><div id="footer">Closing information — copyright, links
<figure> / <figcaption><div class="img-wrap">Image, diagram, or code block with caption

Why Should You Care?

♿

Accessibility (a11y)

Screen readers like NVDA and JAWS use semantic tags as landmarks. Visually impaired users can jump straight to <main>, skipping repeated navigation.

🔍

SEO (Search Engine Optimization)

Search engines reward pages with semantic HTML because it makes content easier to index. The W3C (World Wide Web Consortium) authored HTML5 to encourage this.

🧑‍💻

Developer Experience

Six months from now, <nav> tells you instantly what that block does. <div class="x34-top"> does not. Semantic code is maintainable code.

Code Example — Campus Newspaper

<!-- HEADER: the banner identifying the site -->
<header>
    <h1>Campus Chronicle</h1>
    <p>Delivering campus stories since 2024</p>
</header>

<!-- NAV: primary navigation links -->
<nav>
    <a href="#news">News</a>
    <a href="#sports">Sports</a>
</nav>

<!-- MAIN: the dominant, unique content -->
<main>
    <!-- ARTICLE: a self-contained story -->
    <article>
        <h2>New CS Building Opens</h2>
        <p>The long-awaited wing will feature three web-dev labs...</p>
    </article>
</main>

<!-- ASIDE: secondary content (sidebar) -->
<aside>
    <h3>Upcoming Events</h3>
</aside>

<!-- FOOTER: page-bottom info -->
<footer>
    <p>&copy; 2026 Campus Chronicle</p>
</footer>

Replace structural <div> tags with HTML5 semantic elements. Your pages become accessible, search-engine-friendly, and self-documenting — with zero performance cost.

TOPIC 02

CSS Flexbox

One-dimensional layouts made easy

Flexbox (Flexible Box Layout) distributes items along one axis — either a row or a column. Before Flexbox, centering a div vertically required hacky workarounds. Flexbox solved that with a single line: align-items: center.

Flexbox handles one-dimensional layouts — rows OR columns. Activate with display: flex on the parent. Use justify-content (main axis) and align-items (cross axis) to control distribution. Use flex-wrap: wrap for responsive card grids.

TOPIC 03

CSS Grid Layout

Two-dimensional page structure

CSS Grid is a two-dimensional layout system — it controls both rows and columns simultaneously. While Flexbox arranges items along a single line, Grid is purpose-built for full page layouts: dashboards, magazines, image galleries.

Grid Terminology

TermMeaning
Grid ContainerThe parent element with display: grid
TrackA single row or column
CellThe intersection of one row and one column
AreaA named rectangular group of one or more cells
fr unitFractional unit — "one share of the remaining space"
Gap (Gutter)Space between tracks

Flexbox vs. Grid — When to Use Which?

Use Flexbox When...

  • Content determines size
  • One-axis flow (nav bar, card row)
  • Items should grow/shrink freely

Use Grid When...

  • Layout determines placement
  • Full page structure needed
  • Precise row + column control

The Grid Workflow

/* Step 1: Define Tracks */
grid-template-columns: 260px 1fr 1fr;
grid-template-rows: auto 1fr auto;

/* Step 2: Name Areas — a visual ASCII map */
grid-template-areas:
    "sidebar header  header"
    "sidebar main    stats"
    "sidebar footer  footer";

/* Step 3: Place Items */
.sidebar { grid-area: sidebar; }
.header  { grid-area: header; }

CSS Grid controls both rows and columns simultaneously. Define structure with grid-template-columns/rows/areas, place children with grid-area, and use fr units for flexible sizing. Collapse to single column on mobile with @media queries.

TOPIC 04

Transitions & Animations

Adding motion and life to your pages

A transition smoothly animates a property change when something triggers it (hover, focus, JS class toggle). A @keyframes animation runs on its own — no trigger needed — and can define unlimited intermediate steps.

CSS Transitions

/* Syntax: property | duration | timing-function | delay */
transition: width 0.5s ease-out 0.1s;

/* Common timing functions: */
/* ease — slow→fast→slow (default) */
/* linear — constant speed */
/* ease-in — slow start */
/* ease-out — slow end */
/* ease-in-out — slow start and end */

@keyframes Animations

@keyframes fadeAndSlide {
    0%   { opacity: 0; transform: translateY(20px); }
    50%  { opacity: 0.5; }
    100% { opacity: 1; transform: translateY(0); }
}
.element { animation: fadeAndSlide 1s ease-out forwards; }

TOPIC 05

Responsive Web Design

One site, every screen size

In 2026 your website will be viewed on a 6-inch phone, a 10-inch tablet, a 15-inch laptop, and a 32-inch monitor — sometimes all by the same person in one day. RWD (Responsive Web Design) makes one codebase work everywhere.

The Three Pillars

📐

Fluid Grids

Percentage-based or fr-based sizing instead of fixed pixel widths.

🖼️

Flexible Media

Images and videos that scale with their container, never overflowing.

📱

Media Queries

CSS conditional blocks that apply rules only when the viewport matches a condition.

Common Breakpoints

BreakpointTargetTypical Layout
< 600pxPhone (portrait)1 column, stacked
600px – 1023pxTablet2 columns
≥ 1024pxDesktop3 columns
≥ 1440pxLarge desktop3–4 columns, wider content
⚠️

The Critical Tag: Without <meta name="viewport" content="width=device-width, initial-scale=1.0">, mobile browsers render at ~980px and zoom out, ignoring all your responsive CSS.

Mobile-First Code Pattern

/* BASE: mobile (single column) — no media query needed */
.menu { display: grid; grid-template-columns: 1fr; }

/* TABLET: 600px and wider → two columns */
@media (min-width: 600px) {
    .menu { grid-template-columns: repeat(2, 1fr); }
}

/* DESKTOP: 1024px and wider → three columns */
@media (min-width: 1024px) {
    .menu { grid-template-columns: repeat(3, 1fr); }
}

/* PRINT: remove colours, hide buttons */
@media print {
    body { background: white; }
    .order-btn { display: none; }
}

Use <meta viewport> + mobile-first CSS + @media (min-width: ...) queries to create layouts that adapt to any screen. Write for phones first, then add columns for tablets and desktops. Test with browser DevTools device toolbar (F12).

Advanced HTML5 Forms

Built-in validation — no JavaScript needed

TOPIC 07

CSS Variables & Theming

Maintainable, dynamic style systems

CSS Custom Properties (commonly called "CSS Variables") let you store a value once and reuse it everywhere. Declared with the -- prefix, used with var(). They follow the DRY (Don't Repeat Yourself) principle.

Syntax

/* Declare variables in :root (global scope) */
:root {
    --brand-color: #6c5ce7;
    --spacing-md:  1.2rem;
}

/* Use variables anywhere with var() */
.button {
    background: var(--brand-color);
    padding: var(--spacing-md);
}

/* Theme override — just redefine the variables */
html.dark {
    --brand-color: #e17055;
}

CSS Custom Properties (--name: value / var(--name)) eliminate repetition and enable dynamic theming. Declare in :root for global scope, override in theme classes, and swap themes with a single class toggle on <html>. Every major CSS framework relies on this pattern.
