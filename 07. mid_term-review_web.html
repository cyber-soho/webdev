Comprehensive Content Review

This section breaks down the foundational theory across all six modules. Use the tabs below to navigate between chapters without losing your place.

Chapter 1 — User Experience Design (UXD)

What Is UXD? UXD is the disciplined process of shaping digital products so they deliver meaningful, relevant, and satisfying experiences to the people who use them. It is NOT simply about making something look attractive.

The Five Core Components

  • User Research: Understanding users, their goals, and frustrations. The cardinal rule: you are not your user.
  • Information Architecture (IA): Organizing, structuring, and labeling content so users can find what they need without effort (e.g., a well-organized restaurant menu). Reduces cognitive load.
  • Interaction Design (IxD): Defining how users interact with a product (flows, transitions, feedback).
  • Visual Design: Purposeful aesthetics. Every visual choice must serve the user's goals.
  • Usability Testing: Validating with real users. Testing with 5 users reveals roughly 85% of usability problems (Jakob Nielsen).

Chapter 2 — User Interface (UI)

UI vs. UX: UI = How the product looks (visual layer). UX = How the product feels (complete journey).

Five Core Principles of Effective UI

  • Clarity: Elements communicate purpose immediately.
  • Consistency: Reduces learning burden.
  • Feedback: Visible system response to every user action.
  • Simplicity: Removes elements that don't serve the current goal.
  • Accessibility (a11y): Usable by people with disabilities (legal requirement under ADA/EAA).

Chapter 3 — Usability

ISO 9241-11 Definition: "the extent to which a product can be used by specified users to achieve specified goals with effectiveness, efficiency, and satisfaction in a specified context of use."

Nielsen's Five Pillars of Usability

  1. Learnability
  2. Efficiency
  3. Memorability
  4. Errors (frequency and severity)
  5. Satisfaction

Chapter 4 — The Browser

Browser Engines: Blink (Chrome, Edge, Opera), WebKit (Safari), Gecko (Firefox).

The Rendering Pipeline

  1. Parse HTML → Build DOM
  2. Parse CSS → Build CSSOM
  3. Combine → Build Render Tree
  4. Layout (Reflow) → Calculate positions
  5. Paint → Fill pixels
  6. Composite → Layer elements

Chapter 5 — HTML5 & CSS

Semantic Elements: Replaced generic divs. Improves Accessibility (screen readers), SEO, and maintainability. Examples: header, nav, main, article, section, aside.

Layout Systems

  • Flexbox: One-dimensional layout (row OR column). Uses justify-content (main axis) and align-items (cross axis).
  • Grid: Two-dimensional layout (rows AND columns simultaneously). Introduces the fr fractional unit.

Chapter 6 — JavaScript

Scope: Use let and const for block scope instead of var. Closures allow inner functions to retain access to outer scope variables, enabling private state.

Higher-Order Functions (HOFs)

  • map(): Transforms every element into a new array.
  • filter(): Returns new array of elements passing a test.
  • reduce(): Collapses an array into a single value.
