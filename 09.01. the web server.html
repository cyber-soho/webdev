1. Core Concepts: The Request/Response Cycle

This section explores the fundamental nature of web servers. A web server is simply a programme (or the machine running it) that listens for incoming requests and responds with the correct content. All web communication relies on a strict two-step dance: the Request and the Response. Interact with the analogy below to see this in action.

2. The HTTP Protocol

Here we dive into HyperText Transfer Protocol (HTTP), the text-based language of the web. You will learn about the standard "verbs" (methods) used to tell the server what to do, and the status codes the server uses to report back on the outcome of those actions.

🗂️ HTTP Verbs (Methods)

  • GET
    "Give me this"
    Analogy: Can I see the menu?
  • POST
    "Store this new data"
    Analogy: Place a new order
  • PUT
    "Replace this entirely"
    Analogy: Change my whole order
  • PATCH
    "Update part of this"
    Analogy: Add jalapeños
  • DELETE
    "Remove this"
    Analogy: Cancel my order

🚦 Status Code Explorer

2xx = Good | 3xx = Redirect | 4xx = Client Fault | 5xx = Server Fault

3. Content & Server Technologies

Websites serve two types of content: Static (files sent as-is) and Dynamic (files generated on-the-fly). To handle the massive scale of the modern web, different server software engines have been developed. Below, explore the differences between the "Big Three" servers: Apache, Nginx, and Node.js.

📊 The Big Three Web Servers Comparison

Relative mapping of architectural traits based on historical context and design.

Apache (1995)
Process-per-request. Reliable general-purpose workhorse. Analogy: Sit-down steakhouse.
Nginx (2004)
Event-driven. Solved C10K problem. Great reverse proxy. Analogy: Conveyor-belt sushi.
Node.js (2009)
JS runtime. Event-driven, non-blocking. Real-time apps. Analogy: Custom food truck.

4. Under the Hood: The Pipeline

How does an application actually process a request? It relies on Networking (IPs and Ports to find the app), Routing (directing traffic based on the URL path), and Middleware (a pipeline of functions the request passes through before reaching the final handler).

🔨 Building the Minimal Server

Core pattern in Node.js (10 lines):

require('http');

http.createServer((req, res) => {
    res.statusCode = 200;
    res.end('Hello, World!');
}).listen(3000);
IP = Building (e.g., 127.0.0.1)
Port = Apartment (e.g., :3000)

🚄 Routing Logic

url === '/' ➞ Home Page
url === '/about' ➞ About Page
url === '/api/data' ➞ JSON Data
else ➞ 404 Not Found
