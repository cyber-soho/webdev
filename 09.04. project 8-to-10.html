The Web Server & Node.js Fundamentals

Weight 1 · ~1.5 hrs ▼

Step 1 — Environment Setup (~20 min)

  1. Install Node.js LTS (Long-Term Support). Confirm: node --version and npm --version.
  2. Create your project: mkdir my-backend && cd my-backend && npm init -y.
  3. Open package.json — document in your report what each key field means.

Step 2 — Build a Raw HTTP (HyperText Transfer Protocol) Server (~25 min)

Using only the built-in http module (no Express yet), create a server with three behaviours:

  • GET / → HTML page with your project name (from Modules 1–2).
  • GET /api/status → JSON (JavaScript Object Notation) with server time + Node.js version.
  • Any other URL → 404 Not Found.
// Load the built-in HTTP module — no npm install needed
const http = require('http');
const PORT = 3000;

const server = http.createServer((req, res) => {
    // Route: Home page
    if (req.url === '/' && req.method === 'GET') {
        res.writeHead(200, { 'Content-Type': 'text/html' });
        res.end('<h1>My Project — Home</h1>');

    // Route: API status endpoint
    } else if (req.url === '/api/status') {
        res.writeHead(200, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({
            time: new Date().toISOString(),
            node: process.version
        }));

    // Fallback: 404 for everything else
    } else {
        res.writeHead(404);
        res.end('404 Not Found');
    }
});

server.listen(PORT, () => console.log(\`Running at http://localhost:\${PORT}\`));

Comment every line of your code. This is assessed.

Step 3 — Inspect the HTTP Cycle (~15 min)

Open DevTools → Network tab → visit each route. Document: method, URL, status code, Content-Type header. Take one screenshot of a request/response pair.

Step 4 — Reflection (~10 min)

Write ~1 paragraph: What becomes difficult with the raw http module at 50+ routes? What is missing? (This prepares the ground for Task 9.)

📋
Deliverables: Running raw HTTP server (3 routes), commented code, 1 Network-tab screenshot, reflection paragraph.

Express.js — Building a Real API

Weight 2 · ~3 hrs ▼

Step 1 — Setup (~15 min)

# Install Express as a dependency
npm install express
const express = require('express');
const app = express();

// Middleware: parse incoming JSON bodies automatically
app.use(express.json());

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => console.log(\`Express on :$\{PORT}\`));

Step 2 — CRUD (Create, Read, Update, Delete) Routes (~60 min)

Choose a resource central to your project. Implement five RESTful endpoints:

MethodRouteActionSuccess Code
GET/api/<resource>Return all items200
GET/api/<resource>/:idReturn one item200 / 404
POST/api/<resource>Create item201
PUT/api/<resource>/:idUpdate item200 / 404
DELETE/api/<resource>/:idDelete item200 / 404

Store data in a JavaScript array for now (persistent DB in Task 10). Comment every route handler line-by-line.

Step 3 — Middleware (~30 min)

  1. Logger middleware: prints timestamp + method + URL for every request. Must call next().
  2. Error handler: 4-parameter middleware (err, req, res, next) → returns JSON error + 500 status.
  3. One more middleware of your choice: CORS (Cross-Origin Resource Sharing) via cors, rate limiter, auth token check, etc.

Step 4 — Test Your API (~30 min)

Use curl, Postman, Insomnia, Thunder Client, browser Fetch, or any tool you research. Document ≥3 tests (request → response) with screenshots or terminal output.

Step 5 — Connect Your Front-End (~30 min)

Modify your Module 2 front-end to fetch() data from your Express API. If CORS issues arise, explain what CORS is, why the browser blocks it, and how you fixed it.

Step 6 — Reflection (~15 min)

Compare raw http (Task 8) vs Express: lines of code, middleware ease, what would you still need for production (validation, auth, HTTPS (HyperText Transfer Protocol Secure), rate limiting, file logging, etc.)?

📋
Deliverables: 5 CRUD endpoints, ≥2 custom middleware, ≥3 documented test results, front-end connection, raw-vs-Express comparison. All code commented.

Databases: SQL & NoSQL

Weight 1 · ~1.5 hrs ▼

Step 1 — Choose & Install (~20 min)

Pick one SQL (Structured Query Language) database (SQLite, PostgreSQL, MySQL, etc.) and one NoSQL database (MongoDB, Redis, CouchDB, etc.). Install locally or use a free cloud tier.

Step 2 — Schema / Data Model (~15 min)

For your resource from Task 9, define:

  • SQL: A CREATE TABLE statement (columns, types, primary key, constraints).
  • NoSQL: A document structure (JSON shape with at least one nested field or array).
-- SQL schema example (SQLite)
CREATE TABLE IF NOT EXISTS tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    completed INTEGER DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now'))
);
// NoSQL document structure (MongoDB)
{
    title: "Set up database",
    completed: false,
    tags: ["backend", "week-09"],  // nested array
    created_at: new Date()
}

Step 3 — Wire SQL into Express (~20 min)

Replace the in-memory array for at least one route with a real SQL query. Data should survive server restarts.

Step 4 — NoSQL Experiment (~20 min)

Create a script or route that connects to your NoSQL DB, inserts one document, retrieves all, and prints/returns results.

Step 5 — Compare (~15 min)

In your report, create a comparison table (≥6 criteria): data model, schema flexibility, query language, relationships, scalability, best use case. Justify which database you choose for the rest of your project.

CriterionSQLNoSQL
Data modelTables, rows, columnsDocuments (JSON-like)
SchemaFixed (defined upfront)Flexible (varies per doc)
Query languageSQL (standardised by ISO)Driver methods / DSL (Domain-Specific Language)
RelationshipsForeign keys, JOINsEmbedded docs / manual refs
ScalabilityVertical (bigger server)Horizontal (more servers)
Best forStructured, relational dataUnstructured, rapidly evolving data
