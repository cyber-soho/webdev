Web Application Architectures

Explore the blueprints of web applications. This section breaks down the universal components every web app shares, how they are arranged into tiers, the architectural choice between unified and distributed systems, and the rendering strategies that impact user experience and performance.

The 3 Universal Components

Every web app, without exception, consists of these three parts. The architecture simply defines how they are arranged.

💻

Client

The Front-End. What the user directly sees and interacts with.

⚙️

Server

The Back-End. The processing brain containing business logic.

🗄️

Database

The persistent storage where application data lives securely.

Tier-Based Architectures

Tiers define where the components live. 3-tier is the modern default because adding an App Server in the middle protects data and unlocks independent scaling.

Architecture Where things live
1-TierAll on one machine (Client, Server, DB together).
2-TierClient connects directly to Database.
3-Tier (Winner)Client → App Server → Database. Security, maintainability, scalability.
N-TierMany specialized layers distributed across networks.

Monolith vs Microservices

Start with a monolith. Move to microservices only when the pain of the monolith outweighs the complexity of microservices.

Monolithic

  • One codebase, one deployment.
  • ✅ Simple at first. Good for small teams & tight deadlines.
  • ❌ Painful as it grows.

Microservices

  • Many small, independent services.
  • ✅ Independent teams, languages, deployments. Good for scale.
  • ❌ Network complexity, harder debugging.

Rendering & Navigation Paradigms

Pick based on what your users care about most: immediate initial load and SEO (MPA), or seamless interactive navigation after load (SPA).

MPA / SSR

📄📄

Multi-Page App / Server-Side Rendering

  • First Load: Fast
  • Navigation: Slow (full reload)
  • SEO: Excellent
  • Examples: Wikipedia

SPA / CSR

⚡

Single-Page App / Client-Side Rendering

  • First Load: Slow (downloads JS)
  • Navigation: Very Fast
  • SEO: Tricky
  • Examples: Gmail, Trello
Modern Hybrid: Frameworks like Next.js, Nuxt, and SvelteKit do both, known as Universal Rendering or JAMstack.

Architecture Decision Matrix

Architecture is never about the "best" technology — it is about the trade-offs. The senior developer picks the right trade-off for the right project. Review the core concepts below.

🌐 Web Rules

  • 01

    Always 3 Components

    Client, Server, Database. Pick tier count based on security & scale.

  • 02

    Team Size Dictates Structure

    Small team? Monolith. Large team needing isolated deployments? Microservices.

  • 03

    UX vs SEO

    Pick SPA for interactive user experience post-load. Pick MPA for excellent first paint and SEO.

📱 Mobile Rules

  • 04

    Performance Critical = Native

    Games, AR, heavy hardware access require Swift (iOS) or Kotlin (Android).

  • 05

    Modern Startup Default = Cross-Platform

    React Native or Flutter. ~85% code reuse with near-native speed.

  • 06

    Lightest Weight = PWA

    Manifest + Service Worker. Installable, offline capable, no app store friction.
