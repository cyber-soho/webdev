🔍 Deep Dive: Key Defences

The Threat: Concatenating user input directly into DB queries. A payload like ' OR '1'='1 bypasses logic.

The Defence: Parameterised Queries. Send the query and values separately. The database treats values as data, never code.

SELECT * FROM users WHERE username = ?

The Threat: Rendering unescaped user input in HTML, allowing attackers to run JavaScript in visitors' browsers (Stored, Reflected, or DOM-based).

The Defence: Output Encoding. Convert special characters before inserting. Use safe framework primitives (e.g., {value} in React). Never trust data on the way in; escape on the way out.

Passwords: Never store plaintext, MD5, or SHA-1. Current standard is bcrypt / scrypt / argon2 (deliberately slow hashes).

Sessions: Use cryptographically random IDs. Store in cookies marked HttpOnly, Secure, SameSite=Lax.

MFA: Combine something you know (password) with something you have (app code).

Guarantees: Encryption, Integrity, and Authentication.

Implementation: Free via Let's Encrypt. Force HTTP-to-HTTPS redirects. Use headers like Strict-Transport-Security and Content-Security-Policy.

The 5 Rules of Web-App Security

  1. Validate every input on the server. Client-side is a courtesy, not a defence.
  2. Parameterise every query. No string concatenation, ever.
  3. Escape every output. Especially HTML.
  4. Hash every password with bcrypt. Hash. Never encrypt. Never store.
  5. Wrap everything in HTTPS. Free, automatic, no excuses.

🔬 Anatomy of a Good Test

The AAA Pattern

  • Arrange: Set up inputs and the world state.
  • Act: Call the function exactly once.
  • Assert: Check outputs match expectations.

FIRST Principles

Tests must be: Fast, Isolated (no dependencies), Repeatable, Self-validating (pass/fail), and Timely (written before/with code).

The 5 Rules of Testing

  1. Build a pyramid. Many unit, some integration, few E2E.
  2. Keep tests FIRST. Fast, Isolated, Repeatable, Self-validating, Timely.
  3. Use AAA. Arrange, Act, Assert. One assertion concept per test.
  4. Select by data-testid. Never by CSS class. Stability over beauty.
  5. Run them in CI. A test you run by hand is a test you forget to run.
